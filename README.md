# カット計算アプリ（バカラ上限超過ベットの按分カット）

バカラで「賭け先ごとの上限」を超えたベットを、比例配分でカットして客に返す額を計算する実務ツール。

## どれが本物か（重要）

- **本物のアプリはこの1ファイルだけ：** `baccarat-cut-work.html`
- 単一HTML（CSS・JS・計算ロジック全部入り）。ブラウザで開くだけで動く。
- 過去の重複・旧版は `~/Desktop/ClaudeApps/_旧アーカイブ_2026-06-18/` に退避済み。
  今後アプリを直すときは、**必ずこのリポジトリ内の `baccarat-cut-work.html` を編集**すること。

## テスト（計算が壊れていないかの自動チェック）

お金の計算なので、直したら必ずテストして「答えが変わっていないか」を確認する。

- **かんたん実行：** `テスト実行.command` をダブルクリック → 「✅ 全テスト通過」が出ればOK。
- ターミナルから：
  ```sh
  jsc tests/cut-core.test.js -- baccarat-cut-work.html tests/snapshot.json
  ```
  （`jsc` は `/System/Library/Frameworks/JavaScriptCore.framework/Versions/A/Helpers/jsc`）

テストの中身：
- **不変条件テスト**：ランダムな入力を5000通り作り、「有効ベット合計が上限を絶対に超えない」「各席は最低保証を下回らない」などの“崩れてはいけないルール”を毎回チェック。
- **修正ガードテスト**：後述のバグ修正（最低保証は$100単位）が外れていないかをチェック。
- **スナップショット**：代表9ケースの答えを `tests/snapshot.json` に固定し、以前と同じ答えを出すか比較。

### わざと答えを変えたとき（基準を更新するには）
意図して計算仕様を変えた場合だけ、基準データを作り直す：
```sh
jsc tests/cut-core.test.js -- baccarat-cut-work.html tests/snapshot.json generate > tests/snapshot.json
```

## 既知バグの修正メモ

- **症状**：最低保証を半端な額（例 $210, $50）にすると、$100チップを割らないカット制約のせいで十分に削れず、**有効ベット合計が上限を超える**ことがあった（＝客に上限より多く残る）。
- **原因**：カットは$100単位が基準。保証額が$100の倍数でないと削り切れない組み合わせが生じる。
- **修正**：最低保証の入力を **$100単位のみ** に固定（設定欄 `step="100"` ＋ `validateLG` で$100の倍数チェック ＋ 起動時に保存値を$100へ丸め）。上限ベット・ベット入力（$10単位）は変更なし。
- 100万通りの検証で、$100単位の保証なら上限超過は0件。

## 変更したときの流れ（運用の型）

1. `baccarat-cut-work.html` を編集
2. `テスト実行.command` で ✅ を確認
3. git で記録（例）：
   ```sh
   git add baccarat-cut-work.html
   git commit -m "変更内容を書く"
   ```
4. （必要なら）`git push` でGitHubへ。push前は内容をよく確認する。
